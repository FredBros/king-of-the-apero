À partir de maintenant, nous utilisons la méthode BMAD. Pour chaque demande :

    1- (B) Brain Dump : Analyse l'impact de ma demande sur le code existant.

    2- (M) Map : Propose-moi un plan d'action technique avant d'écrire le code. Attends ma validation.

    3- (A) Action : Ne génère le code que pour une seule 'User Story' à la fois.

    4- (D) Done : Fournis-moi une liste de vérifications (QA) pour tester si la feature fonctionne dans Godot.
